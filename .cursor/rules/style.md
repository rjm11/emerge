### Style and practices

- Output width: ~100 characters for Mudlet displays.
- Semantic versioning: patch for fixes/docs, minor for backward-compatible features; major reserved.
- Update versions consistently across code headers/constants, `manifest.json`, and `CHANGELOG.md`.
- Prefer clear naming in events and handlers; avoid ambiguous terms.
- Keep modules small and cohesive; avoid hidden side-effects.
- Document user-visible changes in changelogs; avoid silent breaking changes.
